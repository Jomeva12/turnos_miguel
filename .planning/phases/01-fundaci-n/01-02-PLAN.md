  <name>Task 1: Configure better-auth server + client, API route, and middleware</name>
  <files>lib/auth.ts, lib/auth-client.ts, app/api/auth/[...all]/route.ts, middleware.ts, .env</files>
  <action>
    1. Create `lib/auth.ts` — better-auth server configuration:
       - Import betterAuth from "better-auth" and prismaAdapter from "@better-auth/prisma"
       - Import prisma from "@/lib/db"
       - Configure: database = prismaAdapter(prisma, { provider: "postgresql" })
       - Enable emailAndPassword: { enabled: true }
       - Configure session: { cookieCache: { enabled: true, maxAge: 5 * 60 } }
       - Export `auth` instance
       - Add BETTER_AUTH_SECRET to .env (generate a random 32-char string)
       - Add BETTER_AUTH_URL=http://localhost:3000 to .env

    2. Create `lib/auth-client.ts` — better-auth client:
       - Import createAuthClient from "better-auth/react"
       - Export authClient with useSession, signIn, signOut hooks
       - This file is used in Client Components

    3. Create `app/api/auth/[...all]/route.ts`:
       - Import auth from "@/lib/auth"
       - Export GET and POST handlers: `export const { GET, POST } = auth.handler`

    4. Create `middleware.ts` for route protection:
       - Check session via better-auth's `auth.api.getSession({ headers: request.headers })` — use request.headers directly (NOT the Next.js `headers()` import, which doesn't work in Edge runtime per research pitfall #3)
       - Public routes: /login, /api/auth/* (all auth API routes)
       - If no session and not public route: redirect to /login (silent redirect, no toast per user decision)
       - If session and on /login: redirect to /planilla
       - Matcher: exclude _next/static, _next/image, favicon.ico, public assets

    5. Run `npx @better-auth/cli generate` to verify auth tables align with Prisma schema. If better-auth needs additional fields on User/Session/Account/Verification models, update schema.prisma and run `npx prisma migrate dev --name auth-fields`. Then re-run seed if the User model changed.

    IMPORTANT: The seed script from Plan 01 created an admin user. Verify the password hashing is compatible with better-auth's sign-in flow. If better-auth uses a different hash format than what seed.ts used, update seed.ts to use better-auth's `hashPassword` utility and re-seed.
  </action>
