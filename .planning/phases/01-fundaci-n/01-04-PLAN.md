<task type="auto">
  <name>Task 1: Dockerfile, docker-compose, and entrypoint script</name>
  <files>Dockerfile, docker-compose.yml, docker-entrypoint.sh, .dockerignore, .env.example</files>
  <action>
    1. Create `.dockerignore`:
       ```
       node_modules
       .next
       .git
       .env
       .env.local
       .planning
       ```

    2. Create `Dockerfile` with three-stage build:
       - Stage 1 (deps): FROM node:20-alpine, WORKDIR /app, COPY package*.json, RUN npm ci
       - Stage 2 (builder): FROM node:20-alpine, WORKDIR /app, COPY --from=deps, COPY source, RUN npx prisma generate, RUN npm run build
       - Stage 3 (runner): FROM node:20-alpine, WORKDIR /app, ENV NODE_ENV=production, ENV TZ=America/Bogota (CRITICAL — per research pitfall #2, dates shift by one day without this), COPY standalone output, static, public, prisma directory, entrypoint script. RUN chmod +x docker-entrypoint.sh. EXPOSE 3000. ENTRYPOINT ["./docker-entrypoint.sh"]
       - Install tzdata in runner stage for TZ support: `RUN apk add --no-cache tzdata`
       - Copy prisma directory (schema + migrations) into runner for migrate deploy

    3. Create `docker-entrypoint.sh`:
       ```bash
       #!/bin/sh
       set -e
       echo "Running database migrations..."
       npx prisma migrate deploy
       echo "Starting server..."
       node server.js
       ```
       IMPORTANT: Use `prisma migrate deploy` NOT `prisma migrate dev` (per research anti-pattern). migrate deploy only applies pending migrations without interactive prompts.

    4. Create `docker-compose.yml` for local testing:
       ```yaml
       services:
         db:
           image: postgres:16-alpine
           environment:
             POSTGRES_USER: postgres
             POSTGRES_PASSWORD: postgres
             POSTGRES_DB: gestion_turnos
           ports:
             - "5432:5432"
           volumes:
             - pgdata:/var/lib/postgresql/data

         app:
           build: .
           ports:
             - "3000:3000"
           environment:
             DATABASE_URL: postgresql://postgres:postgres@db:5432/gestion_turnos
             BETTER_AUTH_SECRET: ${BETTER_AUTH_SECRET:-local-dev-secret-change-in-production}
             BETTER_AUTH_URL: http://localhost:3000
             TZ: America/Bogota
           depends_on:
             - db

       volumes:
         pgdata:
       ```

    5. Create `.env.example` documenting all required environment variables:
       ```
       DATABASE_URL=postgresql://postgres:postgres@localhost:5432/gestion_turnos
       BETTER_AUTH_SECRET=generate-a-random-32-char-string-here
       BETTER_AUTH_URL=http://localhost:3000
       TZ=America/Bogota
       ```

    6. First, catch compile errors quickly: `npm run build 2>&1 | tail -3` — if this fails, fix before proceeding to Docker build.
    7. Test the Docker build: `docker compose build` — must complete without errors.
    8. Test full stack: `docker compose up -d` — both services must start. Wait for app to be healthy, then verify http://localhost:3000 redirects to /login.
    9. After confirming Docker works, run `docker compose down` to clean up.
  </action>
