<business_rules>
<!-- Extracted from PROJECT.md and REQUIREMENTS.md -->

DESCANSOS (GEN-04, GEN-05, GEN-06, GEN-13):
- Each employee gets exactly 4 rest days per month
- Breakdown: 3 ordinary rest days + 1 mandatory Sunday rest
- NEVER assign rest on Saturday (dayOfWeek = 6)
- NEVER assign rest in zona crítica: days 28, 29, 30, 31 of current month AND days 1, 2 of NEXT month
- If employee runs out of valid days for their 4th rest, force it on the next available non-critical, non-Saturday day

ROTACIÓN (GEN-07):
- Employees rotate on alternating weeks: week A = morning shifts, week B = afternoon shifts
- Week parity is based on ISO week number (odd/even)
- Morning = timeSlots starting 7:00; Afternoon = timeSlots starting 14:00 or later

TURNOS PARTIDOS (GEN-08):
- Assign split shifts (timeSlot contains "|") 2 times per week maximum
- Format: "7:00-11:00|11:30-14:30" — two periods separated by pipe
- If area has fewer than 2 split shift templates, assign as many as available

ÁREAS ESPECIALES (GEN-11):
- "Marking" area: only on Tuesdays (dayOfWeek=2) and Thursdays (dayOfWeek=4) — skip other days
- "Valery Camacho" area: only on Wednesdays (dayOfWeek=3) — skip other days
- "Buffet" area: no Sunday assignments (dayOfWeek=7)

COBERTURA (GEN-12):
- Each shift_template has requiredCount — the number of employees needed for that slot
- Algorithm must fill up to requiredCount employees per template slot per day
- If not enough qualified employees available, log a warning

COMODINES (GEN-14):
- shift_templates with isWildcard=true are "comodín" slots
- Assign wildcard shifts as fallback when coverage is needed but no regular template applies
</business_rules>
