```prisma
model Shift {
  id         Int      @id @default(autoincrement())
  employeeId Int
  date       DateTime @db.Date
  timeSlot   String?  // "7:00-11:00" o "7:00-11:00|11:30-14:30" (partido)
  areaId     Int?
  isManual   Boolean  @default(false)
  employee   Employee @relation(...)
  area       Area?    @relation(...)
}

model Area {
  id    Int    @id
  name  String @unique
  color String // hex o clase CSS, ej: "#10b981"
}

model Employee {
  id   Int    @id
  name String
}
```

Formato de timeSlot:
- Turno simple: "7:00-14:00" (un bloque)
- Turno partido: "7:00-11:00|11:30-14:30" (dos bloques separados por "|")
- DESCANSO o null: no mostrar bloque

Las 7 areas del sistema:
General, Buffet, Cosmetico, Domicilio, Electrodomestico, Marking, Varely Camacho

Rango Gantt: 6:00 a 22:00 = 960 minutos = 64 intervalos de 15 min
Ancho total del timeline = 100%; cada minuto = (posicion - 360) / 960 * 100%

  <name>Tarea 2: Componente Gantt horizontal</name>
  <files>components/cobertura/CoberturaGantt.tsx</files>
  <action>
    Crear components/cobertura/CoberturaGantt.tsx como Client Component ("use client"):

    Tipos:
    ```typescript
    interface ShiftBlock {
      employeeName: string;
      timeSlot: string; // puede ser "H:mm-H:mm" o "H:mm-H:mm|H:mm-H:mm"
    }

    interface AreaRow {
      areaId: number;
      areaName: string;
      areaColor: string; // hex color del area
      shifts: ShiftBlock[];
    }

    interface CoberturaData {
      areas: AreaRow[];
    }
    ```

    Props: { data: CoberturaData }

    Logica de posicionamiento en el Gantt:
    - Rango visible: 6:00 (360 min desde medianoche) a 22:00 (1320 min desde medianoche)
    - Total de minutos visibles: 960
    - Funcion timeToMinutes(time: string): number — convierte "7:30" a 450
    - Funcion calcPosition(timeInMinutes: number): string — retorna porcentaje: `${((timeInMinutes - 360) / 960) * 100}%`
    - Funcion calcWidth(startMin: number, endMin: number): string — retorna porcentaje del ancho

    Para cada area (fila):
    - Si el area es "General": renderizar UNA sub-fila por cada shift del area (un empleado por fila)
    - Para el resto de areas: renderizar UNA fila con TODOS los shifts de esa area superpuestos o en sub-filas si se solapan (simplificacion: usar sub-filas si hay mas de un empleado en el area)

    Renderizado del timeline:
    - Contenedor relativo con overflow-x-auto (scroll horizontal si es muy ancho en mobile)
    - Header de horas: columnas cada hora de 6:00 a 22:00 (17 labels) con lineas verticales punteadas
    - Por cada hora, mostrar el label "6:00", "7:00" ... "22:00" en text-xs text-gray-500
    - Marcas de cada 15 min: linea vertical muy tenue (opacity-20)

    Por cada fila (area o sub-fila de empleado en General):
    - Label a la izquierda fijo: nombre del area (o nombre del empleado para General), min-w-[120px] text-sm
    - Barra de fondo: bg-white/5 h-8 rounded relative flex-1
    - Por cada bloque de turno en la fila:
      - Parsear timeSlot: si contiene "|", son dos bloques separados
      - Por cada bloque: div absoluto con left=calcPosition, width=calcWidth, height=100%
      - Color del bloque: usar el areaColor del area (o un color fijo por tipo si no hay area)
      - Opacidad: bg-[areaColor]/70 hover:bg-[areaColor]/90
      - Al hover: tooltip con employeeName + timeSlot (title attribute es suficiente)
      - Turnos nulos o DESCANSO: no renderizar bloque

    Si data.areas esta vacio o no hay shifts en ninguna area: mostrar "No hay turnos registrados para esta fecha."

    Estilos generales: fondo glassmorphism, scroll horizontal con overflow-x-auto en el contenedor del gantt, sticky label column a la izquierda.

    IMPORTANTE: Las areas sin turnos ese dia deben mostrarse de todas formas (barra vacia) para que el usuario vea que no hay cobertura. Incluir las 7 areas incluso si no tienen shifts.

    SIMPLIFICACION PERMITIDA: Para areas con multiples empleados simultaneos, apilar en sub-filas (flex-col en el contenedor de la barra). No es necesario detectar solapamientos exactos — una sub-fila por cada shift del area es suficiente.
