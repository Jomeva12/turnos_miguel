  <name>Tarea 1: API Routes para ausencias (GET, POST, DELETE, limpiar)</name>
  <files>app/api/ausencias/route.ts, app/api/ausencias/[id]/route.ts, app/api/ausencias/limpiar/route.ts</files>
  <behavior>
    - GET /api/ausencias?mes=3&anio=2026 devuelve ausencias donde startDate o endDate caen en ese mes/año, incluyendo employee.name
    - POST /api/ausencias con body {employeeId, type, startDate, endDate} valida type in ["VAC","INC","PER","CAL","DESCANSO"], retorna 201 con el Absence creado
    - POST /api/ausencias con type invalido retorna 400 con {error: "Tipo invalido"}
    - DELETE /api/ausencias/[id] retorna 204 si existe, 404 si no existe
    - POST /api/ausencias/limpiar con body {mes, anio} elimina todas las ausencias cuyo startDate cae en ese mes/año (usando gte/lt de primer dia al ultimo dia del mes)
    - Todos los endpoints verifican sesion activa; retornan 401 si no hay sesion
  </behavior>
  <action>
    Crear app/api/ausencias/route.ts:
    - GET: acepta query params `mes` (1-12) y `anio`. Consulta prisma.absence.findMany filtrando por startDate >= primer dia del mes Y startDate < primer dia del mes siguiente (esto captura ausencias que inician en el mes). Include employee { name }. Retorna JSON array.
    - POST: parsea body, valida type con array TIPOS_VALIDOS = ["VAC","INC","PER","CAL","DESCANSO"]. Parsea startDate y endDate como Date (formato ISO "YYYY-MM-DD"). Crea con prisma.absence.create. Retorna NextResponse.json(absence, { status: 201 }).

    Crear app/api/ausencias/[id]/route.ts:
    - DELETE: parsea params.id como Int. Intenta prisma.absence.delete. Si PrismaClientKnownRequestError con code "P2025" (registro no encontrado), retorna 404. Si exito, retorna new Response(null, { status: 204 }).

    Crear app/api/ausencias/limpiar/route.ts:
    - POST: parsea body { mes, anio }. Calcula startOfMonth = new Date(anio, mes-1, 1) y endOfMonth = new Date(anio, mes, 1). Ejecuta prisma.absence.deleteMany({ where: { startDate: { gte: startOfMonth, lt: endOfMonth } } }). Retorna { deleted: result.count }.

    En todos los routes: importar { auth } from "@/lib/auth" y verificar sesion con getSession antes de operar. Retornar NextResponse.json({ error: "No autorizado" }, { status: 401 }) si !session.

    IMPORTANTE: Las fechas en Prisma con @db.Date deben pasarse como objetos Date de JavaScript (no strings). Usar new Date(dateString) para parsear.
  </action>
