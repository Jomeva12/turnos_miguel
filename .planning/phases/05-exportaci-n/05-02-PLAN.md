  <behavior>
    - deleteShiftsByMonth(year: number, month: number): deletes all Shift records where date is within the month (gte first day, lte last day). Returns { deleted: number }.
    - deleteAbsencesByMonth(year: number, month: number): deletes all Absence records where startDate falls within the month (gte first day, lte last day). Returns { deleted: number }.
    - Both functions validate year (> 2000, < 2100) and month (1-12) and throw Error("Parámetros inválidos") if invalid.
    - Both use the Prisma singleton from lib/db.ts (not new PrismaClient()).
    - Date range: first day = new Date(year, month - 1, 1), last day = new Date(year, month - 1, daysInMonth) where daysInMonth = new Date(year, month, 0).getDate().
  </behavior>
  <action>
    Create `app/actions/cleanup.ts` with the `"use server"` directive at the top.

    Implementation:

    ```typescript
    "use server";

    import db from "@/lib/db";
    import { revalidatePath } from "next/cache";

    function getMonthRange(year: number, month: number) {
      if (year < 2000 || year > 2100 || month < 1 || month > 12) {
        throw new Error("Parámetros inválidos");
      }
      const firstDay = new Date(year, month - 1, 1);
      const lastDay = new Date(year, month - 1, new Date(year, month, 0).getDate());
      return { firstDay, lastDay };
    }

    export async function deleteShiftsByMonth(year: number, month: number) {
      const { firstDay, lastDay } = getMonthRange(year, month);
      const result = await db.shift.deleteMany({
        where: { date: { gte: firstDay, lte: lastDay } },
      });
      revalidatePath("/planilla");
      return { deleted: result.count };
    }

    export async function deleteAbsencesByMonth(year: number, month: number) {
      const { firstDay, lastDay } = getMonthRange(year, month);
      const result = await db.absence.deleteMany({
        where: { startDate: { gte: firstDay, lte: lastDay } },
      });
      revalidatePath("/planilla");
      return { deleted: result.count };
    }
    ```

    Key points:
    - `"use server"` directive is required at the top — this is a Server Action file
    - Use `new Date(year, month - 1, 1)` (local time constructor), NOT ISO string — avoids UTC day-shift in TZ=America/Bogota
    - `revalidatePath("/planilla")` refreshes the planilla after deletion so the page reflects the cleared state
    - deleteMany returns `{ count: number }` in Prisma — use `result.count` not `result.length`
